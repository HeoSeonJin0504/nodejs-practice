MVC란? 
- Model-View-Controller의 약자로, 소프트웨어 디자인 패턴 중 하나이다. 이 패턴은 애플리케이션을 세 가지 주요 부분으로 나누어 관리한다.

1. Model
- 데이터와 관련된 모든 것을 관리
- DB와의 상호작용, 데이터 검증, 비즈니스 로직 등을 처리
- 예를 들어, 사용자의 정보를 저장하고 불러오는 역할

2. View
- 사용자에게 보여지는 화면
- HTML, CSS, JavaScript 등을 사용하여 UI 구성
- 예를 들어, 로그인 페이지나 회원가입 페이지 표시

3. Controller 
- 사용자의 입력을 처리하고, 모델과 뷰를 연결하는 역할
- 사용자가 어떤 행동을 했을 때, 그에 맞는 로직을 실행하고 결과를 뷰에 전달
- 예를 들어, 사용자가 로그인 버튼을 클릭하면 로그인 정보를 모델에 전달하고 결과를 뷰에 보여준다.

[HTTP 상태 코드란?]
- 클라이언트(사용자)가 서버에 요청을 보냈을 때, 서버가 요청을 어떻게 처리했는지 알려주는 번호
- 100번대부터 500번대까지 있으며, 주로 200, 300, 400, 500번대가 많이 사용된다.

[자주 사용되는 상태 코드 정리]

200번대: 성공 (Success)
- 200: 요청 성공 (예: 로그인 성공)
- 201: 새로운 데이터를 만드는데 성공 (예: 회원가입 성공)
- 204: 성공했지만 응답 데이터 없음 (No Content)

300번대: 다른 페이지로 이동, 리다이렉션 (Redirection)
- 301: 영구 이동 (Moved Permanently)
- 302: 임시 이동 (Found)
- 304: 페이지가 변경된 것이 없음 (캐시 사용)

400번대: 클라이언트 오류 (Client Error)
- 400: 잘못된 요청 (Bad Request) - 요청 구문, 데이터 형식 등이 잘못됨
- 401: 인증 필요 (Unauthorized) - 로그인 필요
- 403: 권한 없음 (Forbidden) - 접근 권한 없음
- 404: 페이지를 찾을 수 없음 (Not Found) - 요청한 리소스가 없음
- 409: 충돌 (Conflict) - 리소스 상태에 위반됨 (예: 이미 있는 아이디로 회원가입)

500번대: 서버 오류 (Server Error)
- 500: 서버 내부 오류 (Internal Server Error)
- 502: 게이트웨이 오류 (Bad Gateway)
- 503: 서비스 이용 불가 (Service Unavailable) - 서버 과부하 또는 점검 중
